# MindRules — سیستم خبره آکادمیک برای بررسی گرایش‌های شخصیتی

[English](./README.md) · [فارسی](#)

**دانشجو:** جمال سام خانیان  
**استاد:** دکتر رویا نمیرانیان  
**درس:** سیستم‌های خبره

MindRules یک سیستم خبرهٔ سبک و آموزشی بر پایهٔ قواعد است که با HTML، CSS و JavaScript پیاده‌سازی شده است.  
این پروژه برای اهداف **آموزشی و آکادمیک** طراحی شده است و **ابزار تشخیص بالینی نیست.**

---

## هدف

هدف این سیستم شامل موارد زیر است:

- نمایش چگونگی ساختار پایگاه دانش با استفاده از قواعد.  
- نمایش نحوه عملکرد موتور استنتاج زنجیره پیشرو.  
- دریافت و تحلیل پاسخ‌های کاربر به‌صورت تعاملی.  
- ارائه **توضیح علت نتیجه‌گیری** (Explainability).

این پروژه برای دانشجویان درس **سیستم‌های خبره** و **روش‌های استنتاج هوش مصنوعی** مناسب است.

---

## معماری

سیستم به‌صورت ماژولار طراحی شده و فایل‌ها به صورت زیر تفکیک شده‌اند:

```
MindRules/
├── index.html     # صفحه HTML اصلی، رابط کاربری
├── style.css      # استایل با تم روشن
├── rules.js       # پایگاه دانش با سوالات و قواعد
├── engine.js      # موتور استنتاج و منطق اجرا
├── README.md      # مستندات انگلیسی
└── README.fa.md   # مستندات فارسی
```

### ۱. پایگاه دانش (`rules.js`)

- شامل تمامی **سوالات** پرسیده شده از کاربر.  
- شامل تمامی **قواعد**، که هر قاعده دارای:
  - **شناسه**: ID یکتا  
  - **نام**: برچسب قابل خواندن  
  - **شرط**: تابعی برای بررسی حقایق و فعال شدن قاعده  
  - **توضیح**: علت فعال شدن قاعده

**نمونه قواعد:**

- اگر فرد تنهایی را ترجیح دهد **و** جداشدگی عاطفی مداوم داشته باشد → «شیزوئید گونه»  
- اگر روابط ناپایدار **یا** رفتار تکانشی داشته باشد → «مرزی‌گونه»  
- اگر کمال‌گرایی **و** حساسیت به نظم داشته باشد → «وسواسی»  
- اگر همدلی کم **و** رفتار پرخطر داشته باشد → «ضد اجتماعی»

---

### ۲. موتور استنتاج (`engine.js`)

- از **زنجیره پیشرو** استفاده می‌کند:
  1. جمع‌آوری پاسخ‌های کاربر  
  2. بررسی هر قاعده با حقایق وارد شده  
  3. تعیین قواعد **فعال**  
  4. امتیازدهی به هر سوال (مقیاس 0–2)  
  5. نمایش **نتیجه و توضیح علت**

- شامل بخش **توضیح علت فعال شدن هر قاعده** و نمایش امتیازات هر سوال.

---

### ۳. رابط کاربری (`index.html` + `style.css`)

- **یک صفحه** با دو تب اصلی:
  - **مستندات:** توضیح سیستم، قواعد و معماری  
  - **اجرای سیستم:** فرم تعاملی برای پاسخ به سوالات  

- **سوالات:** هر سوال دارای ۳ سطح (0، 1، 2) نشان‌دهنده شدت ویژگی.  
- **پنل نتیجه:** نمایش
  - الگوهای فعال  
  - توضیح علت هر الگو  
  - جدول امتیازات  
  - خروجی JSON برای استفاده خارجی

- **دکمه‌ها:**  
  - اجرا / بررسی  
  - بازنشانی  
  - کپی JSON

- **تم روشن** و طراحی ساده و خوانا.

---

### ۴. نحوه اجرا

۱. فایل `index.html` را در مرورگر باز کنید.  
۲. برای تمامی سوالات پاسخ مناسب را انتخاب کنید.  
۳. روی **"بررسی و اجرا"** کلیک کنید تا نتیجه نمایش داده شود.  
۴. بخش JSON برای بررسی علت نتیجه و شفافیت استنتاج قابل مشاهده است.
